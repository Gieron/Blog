If you were to print the page outlined above, where would you want page breaks to happen? And how would you write the CSS to make it happen?


Allow pages break anywhere, so that pages may be filled to their maximum

This is the default behavior, so nothing needs to be done. 

// no styling required

Each article should be on its own page

Easy, set the following CSS for each article element (except the first). 

@media print {
    article:not(:first-child) {
        break-before: page;
    }
}

The reason not to set this for the first article is that otherwise the first page will be empty. Note that articles longer than one page will still be broken into pages.


Multiple articles may share a page as long as there is enough room

Also easy, set the following CSS for each article element. 

@media print {
    article {
        break-inside: avoid;
    }
}

Note that articles longer than one page will still be broken into pages, but will always start on a new page.


Page breaks are allowed, but not within paragraphs or directly after headings

Somewhat complicated, but the following CSS will accomplish it. 

@media print {
    h1 {
        break-after: avoid;
    }
    p:first-child {
        break-before: avoid;
    }
    p {
        break-inside: avoid;
        margin-bottom: 1px;
    }
}

In order for page breaks to actually happen there must be some space between adjacent paragraphs, otherwise the paragraphs will be considered connected and always be kept on the same page. Here this is accomplished with a bottom margin, but other separations also work.


The same as above, but it should work in Firefox

Not possible, at least not in the same way. Firefox hasn't implemented break-after: avoid. One potential solution is to rewrite the HTML so that each heading and first paragraph sits inside a div with break-inside: avoid.
